## 系統簡介

**羽球庫存共享小幫手**是一套多球隊共用的羽球庫存管理系統，提供以下核心功能：

- 📦 **庫存追蹤**：即時掌握各球種的剩餘數量
- 📝 **領取登記**：記錄每次練球消耗的桶數與領取人
- 💰 **FIFO 結算**：依先進先出原則計算球費分攤
- 🔒 **多球隊隔離**：每個球隊的資料完全獨立
- 🏸 **開團紀錄**：記錄每場開團的場租、出席名單與利潤

## 領取登記

> 用於記錄每次練球時使用的羽球桶數。

### 步驟

1. 點擊「**領取登記**」按鈕
2. **選擇球種**：從下拉選單選擇此次使用的球種
3. **選擇領取日期**：預設為今天，可回填過往日期
4. **輸入領取人姓名**
5. **輸入桶數**：填入此次實際使用的桶數
6. 點擊「**確認提交**」送出

![領取登記視窗：選擇球種、領取日期、領取人與桶數](/manual/pickup-dialog.png)

### 注意事項

- 當所有球種庫存皆為 **0** 時，「領取登記」按鈕會停用，請先入庫
- 選擇的桶數不能超過該球種的現有庫存
- 領取紀錄可在主畫面的列表中點擊刪除（誤操作時使用）

## 結算試算

> 依 FIFO（先進先出）方式計算指定期間內的球費。

### 步驟

1. 點擊「**結算試算**」按鈕
2. 設定篩選條件（均為選填）：
   - **開始日期 / 結束日期**：限定計算區間
   - **球種篩選**：可勾選一種或多種球種；勾選「全部球種」會自動全選
   - **領取人篩選**：可勾選一位或多位領取人；勾選「全部領取人」會自動全選，反之全部個別勾選後「全部領取人」也會自動勾起
3. 點擊「**開始計算**」查看結果

### 結果說明

| 欄位 | 說明 |
|------|------|
| 區間總成本 | 篩選條件內的球費合計 |
| 球種消耗明細 | 各球種消耗桶數與費用 |
| 使用批次 | 展開後列出該球種依 FIFO 消耗的各入庫批次（使用桶數、進價）與平均單價 |
| 計算方式 | FIFO — 依入庫批次價格依序計算 |

![結算試算結果：區間總成本與各球種消耗明細、使用批次](/manual/settlement-result.png)

## 庫存管理

點擊「**庫存管理**」按鈕開啟「庫存管理中心」，內含四個頁籤：**庫存**、**入庫**、**紀錄**、**球種**。

### 庫存概覽（庫存）

- 以表格列出球種的庫存狀態
- 顯示欄位：品牌／型號、**目前庫存**、累積進貨、累積領取
- 目前庫存低於 5 桶時以紅色標示提醒

![庫存管理中心：庫存概覽頁籤](/manual/inventory-overview.png)

### 入庫作業（入庫）

> 記錄購買新羽球並加入庫存。**需輸入入庫管理密碼（預設：1111）。**

**三步驟流程：**

1. **身份驗證**：輸入入庫管理密碼
2. **填寫資訊**：
   - 選擇球種
   - 輸入數量（桶）
   - 輸入每桶單價（元）
3. **二次確認**：核對資訊後點擊「確認入庫」

> 🔐 密碼驗證通過後 **5 分鐘內**不需重複輸入。
> 💡 若尚未建立任何球種，此頁會提示先「前往建立球種」。

![入庫作業：驗證通過後填寫球種、數量與單價](/manual/inventory-restock.png)

### 入庫紀錄（紀錄）

- 列出所有歷史入庫記錄
- 顯示：日期、球種、數量、單價、總額

![入庫紀錄：歷史入庫清單](/manual/inventory-history.png)

### 球種管理（球種）

- **新增球種**：輸入品牌與型號／名稱
- **首頁顯示切換**：切換鈕標示「**首頁：顯示中 / 首頁：已隱藏**」，用於控制該球種是否顯示在主畫面的庫存卡片（庫存與歷史資料均保留）
- **批量更新歷史進貨單價**：可一次改寫該球種所有歷史入庫的單價（會影響後續 FIFO 成本計算，請謹慎使用）
- **刪除球種**：若該球種已有領取（消耗）紀錄，系統會阻擋刪除並提示改用隱藏

![球種管理：切換首頁顯示／隱藏、編輯與新增球種](/manual/inventory-types.png)

### 球隊管理

進入後會看到球隊清單，每個球隊為獨立的開團紀錄空間，以 PIN 碼保護。

![球隊管理：球隊清單與新增球隊、進入、設定按鈕](/manual/clubs-list.png)

#### 新增球隊

1. 點擊「**新增球隊**」
2. 填寫：
   - **球隊名稱**（如：瘋羽無懼）
   - **隊長 / 負責人**姓名
   - **PIN 碼**（進入球隊時需輸入，請妥善保管）
3. 點擊「**建立球隊**」

#### 進入球隊

1. 點擊球隊列的「**進入**」按鈕
2. 輸入 PIN 碼後點擊「**確認進入**」
3. 驗證通過後即可查看該球隊的活動紀錄

> 💡 PIN 碼驗證狀態僅保留於當前分頁，關閉分頁後需重新驗證。

#### 修改球隊設定

點擊球隊列右側的 ⚙️ 齒輪圖示開啟設定，分為兩步驟：

1. **驗證 PIN**：先輸入該球隊目前的 PIN 碼通過驗證
2. **編輯**：可修改球隊名稱、隊長姓名，或更換 PIN 碼（更換 PIN 為選填）

> ⚠️ 由於需先通過 PIN 驗證才能進入編輯，**PIN 碼一旦忘記，將無法自行從介面重設**，請務必妥善保管。

### 活動紀錄

進入球隊後可查看**活動彙總統計**與所有活動列表，並可依日期篩選、新增、查看或刪除活動。

#### 活動彙總與趨勢圖

活動列表上方會顯示彙總統計，所有數字皆會**隨日期區間篩選連動**：

- **日期區間篩選器**：設定「開始日期 ~ 結束日期」後，彙總卡、趨勢圖、列表與合計皆只計入區間內的活動；有值時可點「清除」還原
- **6 張統計卡**：累計用球（顆）、累計球費、累計收費、累計利潤、平均每場利潤、平均出席（人）
- **每月趨勢圖**：點擊「累計用球 / 累計球費 / 累計收費 / 累計利潤」任一張卡，即可在下方展開該指標的逐月長條圖，再點一次收合
- **桌機合計列**：桌機版列表底部有場租／用球數／球費／收費／利潤的總計列
- **手機結算摘要卡**：手機版於列表上方另有「結算摘要」卡，彙整場租、球費、收費與累計利潤（桌機以合計列取代）

![活動彙總：日期篩選、6 張統計卡與活動列表、合計列](/manual/events-list.png)

點擊統計卡即可展開該指標的每月趨勢圖：

![每月趨勢圖：點擊「累計利潤」卡後展開的逐月長條圖](/manual/events-trend.png)

#### 新增活動

1. 點擊「**新增活動**」
2. 填寫出席名單，可切換兩種輸入方式：
   - **LINE 解析**：貼上 LINE 報名訊息並自動解析（詳見下節）
   - **手動輸入**：可用文字框「一行一人」批次貼上並套用預設費用，或用「姓名 + 費用」逐一新增
3. 填寫場地資訊：
   - 活動日期
   - 場地名稱（選填）
   - 場地數（預設 2）、時數（預設 2）、每小時場租
   - **場租小計**：依「場地數 × 時數 × 每小時場租」即時試算（唯讀）
4.（選填）填寫備註
5. 點擊「**建立活動**」

> ⏳ 若貼上了出席名單，建立時會逐一寫入出席者資料，可能需要約 10 秒，請耐心等候。

![新增活動：LINE 解析／手動輸入分頁與場地資訊](/manual/event-create.png)

#### LINE 訊息自動解析

在「新增活動」視窗切到「**LINE 解析**」分頁，將 LINE 群組的報名訊息貼入文字框，設定「預設費用（元）」後點擊「**解析訊息**」，系統會自動擷取：

- 活動日期（支援 `YYYY/MM/DD`、`MM/DD` 等格式）
- 場地名稱（辨識「場館：／場地：／地點：」）
- 時數（辨識 `18:00-21:00`、`18-21` 等時段並換算）
- **每人費用**（辨識「費用：200」或「費用：$200」，會優先於預設費用）
- 出席名單（依訊息順序排列）

**支援的名單格式：**

```
1.晟
2.傑
3.小豬（免費）
```

- 名單開頭以「數字 + `.`／`．`／`、`」辨識（如 `1.珍妮`）
- 無名字的空白編號（如 `11.`）會自動略過
- 遇到「候補」「——」「🈵」等分隔標記時，會自動截斷後段的候補名單（不會誤砍「報名 16/16｜候補 0」這類統計行）
- 解析後可勾選 / 取消勾選個別出席者，或手動修改姓名與費用
- 頂部「統一費用」可一次套用到所有非免費者
- 點擊「**免費**」可標記為免費出席者（不計入收費）

#### 活動列表欄位說明

| 欄位 | 說明 |
|------|------|
| 日期 | 活動日期 |
| 場地 | 場地名稱 |
| 場租 | 場地數 × 時數 × 每小時場租 |
| 用球數 | 本場用球顆數（手動填入或 FIFO 試算帶入；未記錄則留空） |
| 球費 | 用球成本（手動或 FIFO 試算後填入） |
| 收費 | 已繳費出席者費用合計 |
| 利潤 | 收費 − 球費 − 場租（🔴 正值紅色 / 🟢 負值綠色） |
| 狀態 | ✅ 已結算 / ○ 未結算 |

> 點擊任一列可進入「活動明細」查看出席名單與詳細資訊。

#### 刪除活動

滑鼠移到活動列會出現 🗑️ 刪除按鈕（已結算的活動無法刪除）。

### 活動明細

點擊活動列後開啟明細視窗，可管理出席名單與進行結算。

![活動明細：利潤卡、用球成本、出席名單與結算按鈕](/manual/event-detail.png)

#### 出席名單操作

| 操作 | 說明 |
|------|------|
| 點擊費用欄 | 直接編輯該出席者的費用 |
| 點擊「已繳」按鈕 | 切換繳費狀態（已繳 / 未繳） |
| 點擊「免費」按鈕 | 標記為免費，不計入收費統計 |
| 刪除按鈕 | 移除該出席者（結算後不可刪除） |
| 新增出席者 | 在底部輸入姓名與費用後點擊「新增」 |

**批次操作（右上角）：**

- **統一費用**：一次將相同費用套用到所有非免費者
- **全部已繳**：一鍵標記所有未繳且非免費者為已繳費

**多選批次工具列：**

- 勾選名單左側的核取方塊（或表頭的全選框）選取多位出席者後，上方會出現工具列
- 可對已選取者一次執行：**標記已繳**、**標記未繳**、**刪除**（刪除需確認）
- 點「取消」清空選取；活動結算後這些操作皆停用

#### 用球成本

明細視窗會顯示目前的用球成本、用球顆數與計算模式（先進先出 / 手動）。可透過：

- **先進先出自動試算**：使用 FIFO 試算功能自動計算（詳見下節）
- **手動修改**：展開後直接輸入「金額」與「用球數（顆）」；用球數為選填，留空即記為「未記錄」

#### 結算

確認出席名單與費用無誤後，點擊「**結算此活動**」標記為已結算：

- 結算後活動無法刪除，確保歷史記錄完整
- 結算後仍可查看出席名單與費用明細

---

### FIFO 用球成本試算

在活動明細中，點擊「**FIFO 試算**」區塊，自動從庫存紀錄依先進先出原則計算用球成本。

#### 操作步驟

1. 選擇**球種**
2. 輸入本場使用的**羽球數量（顆）**
   - 系統會自動換算：**12 顆 = 1 桶**
   - 顯示換算結果：`X 顆 ÷ 12 = Y.YY 桶`
3. 點擊「**套用成本**」將計算結果填入用球成本欄位

> 💡 FIFO 試算會依照入庫批次的先後順序與單價，準確計算實際球費。

2. 輸入**新密碼**
3. 再次輸入新密碼**確認**

> 💡 亦可選擇「**停用自訂密碼**」，還原為使用系統預設碼（1111）。

## 常見問題

**Q：忘記入庫管理密碼怎麼辦？**
A：預設密碼為 `1111`。若已修改且忘記，請聯絡系統管理員。

**Q：誤刪了領取紀錄可以復原嗎？**
A：目前無法自動復原，請謹慎操作刪除功能。

**Q：停用球種後庫存資料會消失嗎？**
A：不會。停用只是隱藏該球種，所有歷史紀錄與庫存數量均保留。

**Q：結算試算的費用計算方式？**
A：採用 **FIFO（先進先出）** 原則，最早入庫的球批次優先計入成本，確保費用計算準確。

**Q：多人同時操作會有問題嗎？**
A：系統採用資料庫層級的鎖定機制（SELECT FOR UPDATE），確保領取登記不會發生庫存超賣的情況。

**Q：新增活動時 LINE 訊息解析出現奇怪的名稱（如「12.」）？**
A：這通常是因為訊息中有空白的候補名額（如 `11.` 無填名字）。系統會自動略過空白欄位；若仍出現異常名稱，可在「出席名單預覽」中手動勾選取消或修改。

**Q：開團紀錄的利潤為什麼正值是紅色、負值是綠色？**
A：遵循台灣股市慣例，紅色代表獲利、綠色代表虧損，方便直覺判讀。

**Q：球隊 PIN 碼忘記了怎麼辦？**
A：更換 PIN 前需先通過該球隊目前 PIN 的驗證，因此若 PIN 完全忘記，**無法自行從介面重設**，請聯絡系統管理員協助處理。建議建立球隊時即妥善記錄 PIN 碼。

**Q：已結算的活動可以繼續編輯嗎？**
A：已結算的活動仍可查看出席名單與費用明細，但無法刪除活動，確保歷史記錄完整。
